Les bases

Qu'est-ce qu'un rollup ZK — en mots simples ?

Un rollup ZK, c'est comme avoir un notaire capable de tamponner des milliers de documents d'un coup, en prouvant qu'ils sont tous valides — sans en lire aucun à voix haute.

Vos transactions se déroulent sur une chaîne privée et rapide. Périodiquement, une preuve mathématique est générée qui garantit la validité de chaque transaction. Cette preuve est publiée sur KalyChain — une seule preuve compacte couvrant des milliers de transactions.

Résultat : vous obtenez la rapidité et la confidentialité d'une base de données privée avec la sécurité et l'auditabilité d'une blockchain publique. Le meilleur des deux mondes.

Propriétés clés

Privé — les détails des transactions restent sur votre chaîne, non visibles publiquement
Prouvé correct — les maths garantissent qu'aucune transaction invalide ne peut passer
Immuable — une fois réglés sur KalyChain, les enregistrements ne peuvent plus être modifiés
Rapide — confirmation L2 en ~1s, règlement L1 sur KalyChain en quelques minutes
Souveraineté des données

Vos données de transaction ne quittent jamais votre chaîne

KalyRails déploie chaque chaîne client en mode validium. C'est une distinction cruciale par rapport à un rollup ZK standard, et c'est la plus grande garantie de confidentialité que nous offrons.

Dans un rollup standard, les données compressées de chaque transaction sont publiées sur le L1 public — n'importe qui dans le monde peut les inspecter, pour toujours. En mode validium, seule la preuve cryptographique et la racine d'état sont publiées sur KalyChain. Les données réelles de transaction (montants, ID utilisateurs, horodatages) restent totalement hors du L1 public, conservées par vous et votre infrastructure KalyRails.

Vous obtenez la certitude mathématique complète de la vérification par preuve ZK — chaque batch est prouvé valide — sans exposer une seule transaction client à la vue publique. Les régulateurs obtiennent un accès d'audit via votre explorateur de blocs dédié. Vos concurrents ne voient rien.

Validium vs rollup standard

Architecture sans portefeuille

Comment les utilisateurs transigent sans portefeuilles blockchain

La blockchain traditionnelle exige que chaque utilisateur gère un portefeuille — clés privées, phrases de récupération, tokens de gas. C'est rédhibitoire pour les utilisateurs de mobile money qui peuvent être sur un téléphone basique avec une interface USSD.

KalyRails utilise un modèle opérateur dépositaire. Votre backend détient une seule clé de signature et soumet chaque transaction depuis un unique portefeuille opérateur. Du point de vue de l'utilisateur, rien ne change — il appuie sur « envoyer » dans votre application, et l'argent bouge.

Dans les coulisses, le contrat mobile money suit les soldes sous forme de mapping interne indexé par vos ID utilisateurs existants. Il n'y a pas de portefeuille par utilisateur, pas d'adresse par utilisateur et pas de gas par utilisateur. Votre backend appelle simplement transfer(fromUserId, toUserId, amount) et le contrat fait le reste.

Infrastructure

Ce que nous exploitons pour que vous n'ayez pas à le faire

Chaque déploiement KalyRails inclut une pile d'infrastructure blockchain complète, entièrement gérée et surveillée par notre équipe.

Séquenceur

Exécute les transactions, sert l'API JSON-RPC à laquelle votre backend parle, et regroupe les transactions pour la preuve. Le moteur central de votre chaîne privée.

Prover ZK

Génération de preuves accélérée par GPU utilisant le prover Boojum. Produit des preuves de validité cryptographiques qui garantissent mathématiquement la validité de chaque transaction dans un batch.

Explorateur de blocs

Un explorateur dédié à votre chaîne — indexeur, API et interface web. Vos équipes conformité et les régulateurs peuvent vérifier n'importe quelle transaction à tout moment.

Base de données

PostgreSQL managé avec chiffrement, sauvegardes automatiques et reprise après sinistre. Stocke l'état de votre chaîne et les données de l'explorateur en toute sécurité.

Trésorerie de frais KUSD

Les frais par transaction collectés en KUSD au niveau du contrat intelligent sont versés à une adresse de trésorerie dédiée — retirables vers KalyChain L1 via le coffre de tokens natifs quand vous devez réconcilier les revenus.

Portefeuille opérateur L1

Un portefeuille KLC approvisionné sur KalyChain qui paie les transactions de commit de batch et de vérification de preuve sur L1. Géré et rechargé par l'équipe opérationnelle KalyRails.

Sécurité

Sécurité par les mathématiques, pas par la confiance

Preuves de validité ZK

Contrairement aux rollups optimistes qui supposent les transactions valides et s'appuient sur une fenêtre de contestation de 7 jours, les rollups ZK prouvent la justesse en amont. Chaque batch est vérifié mathématiquement avant règlement. Il n'y a aucune fenêtre pour la fraude.

Pile open-source

KalyRails est entièrement construit sur le ZK Stack open-source de ZKsync. La base de code a subi plus de 10 M$ d'audits de sécurité, plus de 35 audits externes et exploite un programme de bug bounty payant jusqu'à 1,1 M$. C'est du code éprouvé.

Privé par conception

Votre chaîne est à permissions — seul votre backend soumet des transactions. Les détails des transactions ne sont visibles sur aucune blockchain publique. Seules les preuves ZK compactes sont publiées sur KalyChain pour vérification.

Règlement infalsifiable

Une fois qu'une preuve est vérifiée sur KalyChain, l'enregistrement de la transaction est permanent et ne peut être altéré par personne — ni par nous, ni par vous, ni par un acteur malveillant. Les mathématiques rendent cela impossible.
